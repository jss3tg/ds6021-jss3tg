10 Question 4

x <- c(0.22, 0.37, 0.57, 0.66, 0.03, 0.92, 0.77, 0.73, 0.11, 0.40)
y <- c(2.14, 1.89, 1.99, 1.88, 2.26, 2.11, 1.92, 1.98, 2.47, 2.10)

predict_3nn <- function(x_new, x_train, y_train) {
  sapply(x_new, function(xi) {
    dists <- abs(x_train - xi)
    nearest_idx <- order(dists)[1:3]
    mean(y_train[nearest_idx])
  })
}

x_grid <- seq(min(x), max(x), length.out = 500)
y_pred <- predict_3nn(x_grid, x, y)

plot(x, y, pch = 16, col = "steelblue", 
     xlab = "x", ylab = "y", 
     main = "3-Nearest Neighbor Prediction")

lines(x_grid, y_pred, col = "tomato", lwd = 2)

11 Question 5

The “largest model” that has an estimate for every covariate combination can be a poor choice because although it can fit the training data, each estimate is based on its own cell. There is no share of information between cells so in a real case scenario where there may be unknown cells, there is no way to create an estimate for these cells in a largest model.

This ties to information sharing in the way that there needs to be some form of information that is shared between cells to create a rounded out estimate for each cell.

If the largest model had to be the reasonable choice, every cell would need to have some form of an estimate where each cell genuinely stands as its own. Each cell would essentially be independent to the point where the age or blood pressure of someone would not be the same as someone with a year difference.
